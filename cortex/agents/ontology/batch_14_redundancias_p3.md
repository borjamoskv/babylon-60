# MATRIZ 4: REDUNDANCIAS ACTIVAS (MITIGACIÓN C5) (041 - 060)

| ID | Redundancia C5 | Función Topológica | Riesgo Mitigado | Coste (Overhead) | Dependencias |
|----|----------------|--------------------|-----------------|------------------|--------------|
| RED-041 | Verificación de Suma de Comprobación (Checksum) | Archivos guardados o transmitidos (modelos/pesas) se verifican contra un SHA-256 pre-aprobado. | Manipulación interceptada (Man-In-The-Middle) o corrupción de disco silenciosa (Bit-rot). | CPU mínima durante I/O (Streaming Checksum) | `hashlib` nativo |
| RED-042 | Proxy Cíclico Inverso (Zero-Trust RPC) | Toda RPC externa (MCP/HTTP) debe ser iniciada desde dentro del núcleo, prohibiendo conexiones entrantes globales. | Secuestro de puertos locales, ataques CSRF en dev-servers locales. | Latencia (Polling) vs (Pushing) | `aiohttp` / Nginx local |
| RED-043 | Cuarentena de Fallback (Graceful Degradation) | Si el Swarm entero colapsa, CORTEX entra en modo de "Supervivencia de Solo Lectura" (Read-Only). | Inserciones y comandos mutantes destrozando la base de datos en pánico. | Operativa limitada (Cero Writes) | Manejadores de Estado (FSM) |
| RED-044 | Detección de Colisiones de Nombres (Namespace Jail) | El orquestador aísla dinámicamente módulos con nombres colisionantes cargándolos bajo alias absolutos (`sys.modules` patching). | `ImportError` o secuestro lógico por dos subagentes creando `utils.py`. | Complejidad de Reflection y metaclasses | `importlib` profundo |
| RED-045 | Persistencia en Disco Doble Búfer (A/B) | El archivo de configuración crítico se escribe en `.config.new`, se sincroniza y luego se mueve atómicamente a `.config`. | Caída eléctrica truncando la escritura del archivo crítico. | 1.5x IOPS por guardado (Write + Rename) | Operaciones de File System Atómicas |
| RED-046 | Shadow Agents (Agentes Sombra) | Agentes operando en memoria local replicando el output del Agente P0 en paralelo para verificar si la LLM alucina. | Falsos positivos en auditorías de código Críticas. | Multiplicador 2x de Token Cost y Latencia | Router LLM |
| RED-047 | Limpiador Pasivo (Scrubber Demon) | Un script crontab local purga diariamente los tokens y logs excedentes >30 días. | Saturación del filesystem de logs masivos generados por el Swarm. | Carga CPU asíncrona menor | Demonios OS (Launchd/Crond) |
| RED-048 | Inyección de Identidad Inmutable (Sigstore) | Los commits C5-REAL del núcleo se firman digitalmente y sin claves (keyless signature). | Gaslighting histórico de repositorios P2P fusionados. | Interacción externa con Sigstore (Red) | `sigstore` Python library |
| RED-049 | Recuperación de Redundancia de Anillo (Ring Buffer Log) | La memoria episódica usa un Ring Buffer fijo. Al llenarse, sobrescribe lo antiguo (FIFO natural). | Fugas de memoria infinita en historiales de chat. | Tamaño de búfer limitado matemáticamente (Truncado) | Estructura `collections.deque` |
| RED-050 | Mutex Distribuido Epímero | Bloqueo lógico con tiempo de vida (TTL) en Redis/SQLite para prevenir Race Conditions entre subagentes paralelos. | Inanición (Starvation) y bloqueos permanentes. | Latencia de Set-NX de la DB | Lockers distribuidos |
| RED-051 | Fallback de Embeddings (Local vs Cloud) | Si la inferencia MLX local falla por OOM, se desvía temporalmente al modelo de Embeddings en la Nube (API). | Congelamiento local al analizar PDFs de 1GB. | Salida a Internet / Latencia externa extra | Router de Inferencia híbrido |
| RED-052 | Precalentamiento de Caché (Warm-up JIT) | Tareas en segundo plano vectorizan los archivos recién creados antes de que el usuario haga una búsqueda. | Búsqueda inicial excesivamente lenta. | Consumo energético de fondo anticipado | Hilos de demonio Watchdog |
| RED-053 | Validadores Termodinámicos de Prompt | Antes de disparar a un LLM, un analizador regex descarta el prompt si es >90% código repetido innecesario. | Consumo hipertrófico e ineficiente de tokens P0. | Tiempo CPU de Regex / Hashing de párrafos | `tiktoken` u optimizadores locales |
| RED-054 | Interrupción Controlada de Flujo (Yield) | Todo algoritmo pesado de indexado P2P hace un `asyncio.sleep(0)` en su loop principal. | Paralización total del Event Loop (Starvation a otras tareas asíncronas). | Incremento del tiempo total de la tarea pesada | Diseño Coroutine amigable |
| RED-055 | Clonado de PTY Determinista | Se guarda el buffer del shell interactivo (ASCII) a un archivo plano de snapshot en tiempo real. | Un agente corrompe un SSH interactivo y no sabemos qué prompt falló. | Espacio en disco para capturas de log bruto | Pexpect / Tmux log pipeline |
| RED-056 | Estratificación de Permisos (Sub-Scopes) | Los subagentes no heredan el JWT/API Key raíz; se les emite una versión truncada de un solo uso o limitado a /scratch. | Fuga lateral de credenciales por Jailbreak. | Complejidad Oauth / Emisión de micro-tokens locales | Vault Credenciales C5 |
| RED-057 | Re-intento Geográfico Diferido | Si un proveedor Cloud falla, el request se lanza a la API en otra región (Europa vs US). | Caídas completas de regiones cloud. | Costo de red intercontinental / Retraso | Proxies o DNS dinámicos |
| RED-058 | Sandbox de Grafo RAG | El subgrafo recuperado para el contexto LLM se encapsula y purga de toda directiva ejecutiva antes de inyectarlo. | Prompts Inyectados en documentos RAG (Confused Deputy). | Computación de sanitización semántica | Detector de Directivas (Taint Analyzer) |
| RED-059 | Asignación Lenta de Memoria (Lazy Allocation) | Objetos masivos de memoria compartida (MMap) solo se abren al ser referenciados, nunca al inicio. | Latencia excesiva de arranque. | Microlatencia agregada durante el runtime medio | Funciones factoría diferidas |
| RED-060 | Resonancia BFT (Múltiples Prompts Aleatorios) | Para auditar código crítico, se enmascaran las variables antes de enviar a 3 LLMs distintas y se fuerza al consenso. | Sesgo del modelo (Bias) que ciegamente "aprueba" todo. | Trabajo preparatorio masivo / Gasto de tokens altísimo | Herramientas de AST Obfuscation (Nombres Aleatorios) |
