# MATRIZ 3: ANTIPATRONES ESTOCÁSTICOS (041 - 060)

| ID | Antipatrón | Disfunción Causal | Señal de Presencia | Impacto en Robustez | Refactor (Alternativa) |
|----|------------|-------------------|--------------------|---------------------|------------------------|
| ANT-041 | Mutabilidad Oscura | Clases y estructuras de datos mutables pasadas por referencia sin inmutabilidad (`dataclasses` / `frozen=True`). | Modificación colateral accidental en hilos paralelos. | Side-effects impredecibles | Usar Pydantic con `frozen=True` o NamedTuples. |
| ANT-042 | Bucle LLM en Caliente | Llamar a una LLM dentro de un bucle `for` o `while` infinito (1 a 1). | `manage_task` o loops forzando facturación masiva. | Muerte Térmica (Quema de Tokens) | Batching Masivo (1 Prompt = N Ítems procesados). |
| ANT-043 | Dependencia Estricta de Versión Menor | `paquete==1.2.3` en vez de `>=1.2.3, <2.0`. | Conflictos de resolución de PIP infinitos (Dependency Hell). | Incapacidad de integrar Skills | Versionado Semántico Relajado / Lockfiles precisos. |
| ANT-044 | Sincronización Estricta P2P | Requerir que todos los nodos del Swarm respondan al mismo tiempo antes de avanzar (Barrier estricta). | El nodo más lento dictamina la velocidad máxima de todo el enjambre. | Latencia Global (Starvation) | Arquitecturas Event-Driven Asíncronas (Pub/Sub). |
| ANT-045 | Uso de Magic Strings | Pasar comandos o claves de diccionarios en crudo `"clave"` dispersas en 100 archivos. | Errores de tipeo imposibles de refactorizar en bloque. | Corrupción Causal | Uso de Enums tipados o Constantes Centralizadas. |
| ANT-046 | Fallo Silencioso por Indentación | Tratar un error de lógica de negocio o indentación como si fuera un simple warning. | Código ejecutando funciones en el namespace erróneo (OOB). | Ejecución Estocástica | Linter estricto + Fall Fast en tiempo de parseo. |
| ANT-047 | Refactor Cosmético | Cambiar nombres de variables (ej. `x` -> `data_value`) sin modificar arquitectura ni optimizar nada. | Diffs masivos que ocultan bugs y rompen el blame del Ledger. | Pérdida de Historial | Prohibición Termodinámica (Mantener nombres a menos que haya mejora algorítmica). |
| ANT-048 | Inyección de Bloqueos en Init | Conectarse a bases de datos o hacer llamadas de red en el método `__init__` de una clase. | Instanciar objetos bloquea la aplicación; tests unitarios imposibles de correr sin DB real. | Latencia e Inestabilidad | Separar inicialización en métodos `async def connect()`. |
| ANT-049 | Aserción Pobre | Usar `assert x` sin un mensaje detallado o peor, depender de `assert` en producción (donde Python `-O` los ignora). | Excepciones vacías `AssertionError`. | Debugging imposible | Uso de `ValueError` explícito u Open Exceptions. |
| ANT-050 | Despliegue de Código Muerto | Dejar funciones deprecadas, comentarios gigantes de código comentado en producción. | Llenado pasivo del contexto del LLM al leer los archivos. | Anergía Cognitiva | Borrado Inmediato (Git preserva el historial, no el código comentado). |
| ANT-051 | Exposición de Estado Crítico | Variables de instancia de una clase accedidas y modificadas directamente sin Propiedades (`@property`). | Des-sincronización de estado interno y eventos. | Mutabilidad Oscura | Inmutabilidad de estado, Getters con emisión de eventos. |
| ANT-052 | Intercepción de Señal Generalizada | Atrapar `Exception` o `BaseException` en el nivel más alto del enjambre ocultando la muerte térmica. | Sistema que "funciona" pero no procesa nada. | Degeneración Epistémica | Atrapar solo subclases específicas de CORTEX. |
| ANT-053 | Multi-Threading LLM Falso | Intentar acelerar llamadas de OpenAI/Anthropic usando hilos en Python pero bloqueando por el GIL o límites de Tasa API. | Falsa concurrencia, 429 Too Many Requests. | Estrangulamiento (Throttling) | `asyncio.gather` con un Semaforo (Limiter) asíncrono. |
| ANT-054 | Construcción Pobre de Rutas | Unir rutas de directorios usando strings en crudo (`dir + "/" + file`) cruzando OS (Windows/Mac). | `FileNotFoundError` en Linux/Mac al importar código de Windows. | Entropía Cruzada | Uso estricto de `pathlib.Path` para topología determinista. |
| ANT-055 | Adivinación de Módulos (Wildcard Imports) | Uso de `from module import *` en archivos principales del Kernel. | Contaminación masiva de namespace y sobreescrituras silenciosas (Shadowing). | Conflictos impredecibles | Imports explícitos y restrictivos. |
| ANT-056 | Configuración Codependiente | Configurar un módulo de IA asumiendo que otro módulo ya inicializó su logger. | Errores si el orden de inicialización cambia en el futuro. | Entrelazamiento Cuántico | Inyección de Dependencias Limpia (Desacoplado). |
| ANT-057 | Reinvención del Gestor de Contexto | Intentar manejar apertura y cierre de recursos (archivos/db) con `try/finally` explícitos por doquier. | Fugas de Memoria / FDs dejados abiertos. | Saturación de File Descriptors | Uso sistemático de `with` y `async with`. |
| ANT-058 | Hardcoding de Rutas Maestras | Escribir `/Users/borja/` estáticamente en scripts. | Falla inmediata en Docker u otro usuario del equipo. | Portabilidad cero | Relatividad mediante `Path(__file__).parent` o variables de entorno base. |
| ANT-059 | Abuso de Dicionarios no Estructurados | Retornar `dict` masivos con miles de claves dinámicas en vez de Pydantic Models. | Falta de Autocompletado, Typos en las claves indetectables hasta Runtime. | Tipado Inexistente (Any-Slop) | `BaseModel` estricto en cada borde del sistema (Boundary). |
| ANT-060 | Loop Infinito por Retry Mal Formado | Intentar reconectar un websocket o DB indefinidamente sin delay o backoff tras una falla dura de red. | CPU al 100%, parálisis de disco (Log Spam). | Tormenta de Hooks / Asfixia | Backoff Exponencial Obligatorio (Jitter). |
