# MATRIZ 3: ANTIPATRONES ESTOCÁSTICOS (021 - 040)

| ID | Antipatrón | Disfunción Causal | Señal de Presencia | Impacto en Robustez | Refactor (Alternativa) |
|----|------------|-------------------|--------------------|---------------------|------------------------|
| ANT-021 | Adivinación de Tipos (Any-Slop) | Utilizar `Any` indiscriminadamente en tipado estático ignorando Mypy/Pyright. | Fallos en linters o bugs de tipado dinámico runtime. | Caos de mantenimiento (Debt) | Tipado estricto `Generic[T]`, `Protocol`, Pydantic. |
| ANT-022 | Silenciamiento Global de Linter | Poner `ignore = ["ALL"]` en vez de corregir la arquitectura subyacente. | Ausencia de Errores pero codebase incomprensible. | Pudrición Estructural Oculta | Arreglo específico o inyección de `# noqa: X` puntual. |
| ANT-023 | Invocación Ciega (YOLO-Push) | Comitear y pushear sin haber ejecutado las suites de test o un run manual mínimo. | CI/CD o GitHub Actions rompiendo inmediatamente. | Latencia (Espera de Fix + Repush) | Protocolo Obligatorio C5-REAL `QA` / `health-cortex`. |
| ANT-024 | Asincronía Síncrona | Forzar que una función `async` sea llamada con bloqueos de OS sin `run_in_executor`. | Bloqueo absoluto del Event Loop de CORTEX. | Caída del motor (No responde) | Paralelismo correcto (Thread/Process Pool). |
| ANT-025 | Duplicación de Singletons | Instanciar múltiples bases de datos SQLite en hilos paralelos abriendo nuevos fd cada vez. | `database is locked` / Corrupción de DB WAL. | Falla termodinámica | Uso de un Pool centralizado o Conexión Singleton P2P. |
| ANT-026 | Log-Spamming Vacío | Imprimir a consola variables masivas (`print(big_json)`) sin truncamiento o estructura. | Terminal ilegible / Consumo inútil de I/O de disco. | Pérdida del vector cognitivo | Rich logging / Truncar outputs > 1KB. |
| ANT-027 | Mocking Absoluto | Crear suites de tests donde todo es un `MagicMock` y no se prueba la lógica real. | Tests pasan 100% pero la integración falla catastróficamente. | Falsa Seguridad (Ilusión) | Tests de Integración reales con bases de datos en memoria. |
| ANT-028 | Sobre-abstracción de Carpintería | Crear 6 capas de abstracción para una función que solo lee un archivo de texto. | Código inexplorable, sobre-ingeniería innecesaria. | Pérdida de Exergía (R9 Override) | Diseño Funcional y Directo (KISS C5-REAL). |
| ANT-029 | Configuración Estática Inflexible | Variables de entorno harcodeadas en constantes Python globales (ej. `PORT = 8000`). | Código no portable / No testable en paralelo. | Conflictos de puerto / Multi-tenant fallido | Uso de Pydantic `BaseSettings` con `.env`. |
| ANT-030 | Polling Adivinatorio de Filesystem | Hacer bucles de lectura para ver si un archivo cambió en vez de usar eventos de OS. | `time.sleep(1)` en bucle infinito / 100% CPU usage. | Calentamiento (Anergía) | Uso estricto de `watchdog` (kqueue/epoll inotify). |
| ANT-031 | Reinvento de la Rueda Criptográfica | Implementar un algoritmo HASH o firma personalizado en vez de usar librerías estándar. | Vulnerabilidades de Side-Channel / Fuga de bits. | Riesgo Criptográfico Catastrófico | Uso estricto de `cryptography` (libsodium/Argon2). |
| ANT-032 | Memoria Epistémica Olvidadiza | Un agente no revisa la memoria (Ledger) y pregunta de nuevo la misma ambigüedad. | Pregunta duplicada ("¿Qué framework uso?"). | Fricción para el Operador (Green Theater) | `ANAMNESIS-Ω` / Consulta al `CORTEX-Persist`. |
| ANT-033 | Refactor Quirúrgico Incompleto | Cambiar el nombre de una clase pero olvidar actualizar sus referencias en el resto del monorepo. | `NameError` masivos tras un "éxito" reportado por LLM. | Rompimiento local global (Blast Radius) | `grep_search` global + Sustitución en todo el AST. |
| ANT-034 | Error Tragado (Silent Catch) | Un bloque `except Exception: pass` que oculta el fallo causal base. | Comportamientos misteriosos difíciles de rastrear. | Debugging Termodinámico infinito | `logger.exception()` / Rethrow / Fail Fast. |
| ANT-035 | Entropía de Nombres (Naming Slop) | Nombres genéricos como `data_processor`, `utils_helper`, `do_stuff()`. | Imposibilidad de construir ontologías semánticas útiles. | Pérdida de Isomorfismo | Naming C5-REAL: Específico, Causal y Exhaustivo. |
| ANT-036 | Prompts Desacoplados | Instrucciones en código que dicen "Sé amable y útil" a un agente de ejecución destructiva. | Bipolaridad del subagente / Reducción de rendimiento. | Output inconsistente (Baja calidad) | System Prompts brutales C5-REAL (Sin filtro). |
| ANT-037 | Dependencia Fantasma | Importar un módulo en Python que funciona localmente por un `pip install` residual pero falta en `pyproject.toml`. | Entorno limpio fallido (Falla en CI o nuevo deploy). | Ilusión de Completitud | Testeo periódico en Sandbox aislado (Docker). |
| ANT-038 | Commits Multipropósito | Hacer un único commit masivo de 5,000 líneas que añade 3 features y arregla 2 bugs. | Imposibilidad de hacer Revert limpio si algo falla. | Granularidad de Rollback destruida | Sagregación Atómica: Un Commit = Una Hipótesis Causal. |
