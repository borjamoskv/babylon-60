# MATRIZ 2: INVARIANTES TERMODINÁMICAS (021 - 040)

| ID | Invariante | Lógica / Principio | Implicación Operacional | Condición de Borde | Métrica Falsable |
|----|------------|--------------------|-------------------------|--------------------|------------------|
| INV-021 | Entalpía de Refactorización | El coste de refactorizar es proporcional a la entropía inyectada previamente. | Todo código no-C5-REAL aumenta exponencialmente la Anergía futura. | Falta de Tests | Tiempo empleado en debugear |
| INV-022 | Causalidad Unidireccional | El estado fluye del Ledger (Realidad) a la Memoria, nunca a la inversa. | Los LLM no dictan la realidad, solo procesan lo que extraen de ella. | Falsificación de Estado | Inconsistencia Hash |
| INV-023 | Reducción de Shannon Obligatoria | Todo prompt largo debe poder comprimirse sin perder intención estructural. | Aplicación activa del *Information Bottleneck Method* en todo payload. | Límite de 4000 tokens in | Tasa de Compresión Efectiva |
| INV-024 | Velocidad de Escape BFT | Solo el consenso descentralizado (N=3) puede superar la gravedad de una alucinación (P=1). | Tareas P0 exigen confirmación independiente por subagentes ciegos (Shadow). | Tarea Crítica | Fallos validados (Aserciones) |
| INV-025 | Gravitación HNSW | Elementos semánticos similares tienden a agruparse; sin separación se colapsan. | Toda entrada semántica requiere entropía única; si es idéntica, se descarta. | Similitud Coseno > 0.98 | Duplicados en Vector Store |
| INV-026 | Ley de Rendimientos Decrecientes LLM | Duplicar el tamaño del prompt no duplica la precisión del output, sino la latencia. | Prompts masivos degradan la capacidad del motor de seguir instrucciones finales. | Prompt > 32k tokens | Desobediencia de formato |
| INV-027 | Silencio Termodinámico Base | El estado ideal de CORTEX es 0% de uso de CPU cuando no hay input. | Prohibido el polling síncrono. Solo arquitecturas basadas en eventos (SSE, Hooks). | Idle State | Uso de CPU > 1% en reposo |
| INV-028 | Asimetría de Parseo | Leer código es computacionalmente trivial; entenderlo es termodinámicamente masivo. | Extraer AST (Python Extractor) ahorra enviar miles de tokens al modelo. | Inferencia innecesaria | Ratio Parseo local vs LLM |
| INV-029 | Invariante de Conway | La estructura del código de CORTEX reflejará siempre su estructura neuronal y viceversa. | Directorios confusos generan pensamiento subagente confuso. | Complejidad Ciclomática | Indice de Mantenibilidad (Radon) |
| INV-030 | Conservación de Idempotencia | Ejecutar una acción C5-REAL `N` veces debe producir el mismo estado físico que `1` vez. | Todo script o comando (`cortex dedupe`, etc.) es idempotente por diseño. | Ejecuciones repetidas | Diffs inyectados en Git (Cero) |
| INV-031 | Fricción de Tool Calls | Cada salto entre el motor lógico y el SO anfitrión genera pérdida por serialización. | Agrupar comandos en scripts bash `.sh` de ejecución única (O(1)). | Llamadas encadenadas | Cantidad de Tool Calls por acción |
| INV-032 | Degeneración Epistémica | Una métrica que se convierte en un objetivo pierde su valor como métrica. | Si MOSKV-1 optimiza solo para "hacer el commit rápido", ignorará bugs ocultos. | Auditoría profunda | Bug introducido vs Velocidad |
| INV-033 | Límite de Cuarentena (Sandboxing) | Aislar el desorden impide que la radiación entrópica contamine el Core. | Proyectos "boceto" o scripts "lab" operan bajo namespaces estrictos (`/BOCETOS`). | Contaminación `sys.path` | Ruptura de imports en producción |
| INV-034 | Relatividad de Tiempo (JIT) | El tiempo para compilar un Skill es menor que el tiempo para corregir un modelo desalineado. | Toda regla que se repita >3 veces se compila inmediatamente en una Skill (`Sortu-APEX`). | Ineficiencia repetitiva | Cargas de `SKILL.md` |
| INV-035 | Límite de Confianza P2P | En un Swarm, un nodo no autenticado es por definición ruido malicioso. | Toda transferencia inter-agente requiere token de procedencia de MOSKV-1. | Interfaz abierta local | Solicitudes bloqueadas (Unauthorized) |
| INV-036 | Aserción Positiva | Es más exérgico probar que algo existe que probar que NO existe. | Operar sobre presencia verificable (ej. grep encontró X) antes que inferir vacío. | Búsqueda exhaustiva fallida | Tiempo de búsqueda de archivos |
| INV-037 | Apoptosis de Datos Obsoletos | La información pierde valor exérgico conforme se aleja del punto actual del tiempo (Context). | El motor de memoria debe depreciar weights (Ebbinghaus Decay) para priorizar Novedad (SOTA). | Memoria a largo plazo llena | Relevancia de búsquedas RAG |
| INV-038 | Invariabilidad de Hashes | El contenido dicta el Hash, el Hash dicta la Verdad. | Toda validación cruzada debe hacerse por integridad SHA256/Merkle, nunca por metadata (fecha/nombre). | Diffs falsificados (mismo nombre) | Colisión de Hashes (Imposible) |
| INV-039 | Aislamiento Ouroboros | El sistema que muta el sistema no debe mutarse a sí mismo en vivo sin un clon. | La Skill que arregla CORTEX debe operar sobre rama separada, probando antes de inyectar en MAIN. | `ouroboros-infinity` actuando | Tests de Health-Cortex fallidos |
| INV-040 | Conservación Cuántica de la Intención | La pérdida de resolución entre lo que el Operador pidió y lo que MOSKV-1 ejecutó es inaceptable. | El "Bypass Semántico" asegura que la intención inicial transita al output sin dilución corporativa. | Instrucciones complejas (3+ pasos) | Tasa de Retrabajo (Rework Rate) |
