| ID | Antipatrón | Disfunción Causal | Señal de Presencia | Impacto en Robustez | Refactor (Alternativa) |
| :--- | :--- | :--- | :--- | :--- | :--- |
| ANT-001 | Error Silencioso Ciego (Blind Swallow) | Bloque `except` genérico que descarta la excepción sin trazar el origen ni propagarla. | `except Exception:` sin re-lanzar (`raise`) ni emitir al Ledger. | Crítico (Datos corruptos silenciosos). | Capturar excepciones específicas, propagar fallos no manejables. |
| ANT-002 | Singleton de Estado Mutable | Clase instanciada globalmente con variables internas que múltiples workers alteran de forma síncrona. | Módulos importando la misma instancia global para alterar diccionarios o listas. | Alto (Carreras de condición impredecibles). | Inyección de Dependencias; Redispacher de estado (Actor Model). |
| ANT-003 | Hardcoding de Configuraciones Dinámicas | Claves de API o rutas específicas fijadas en código en lugar de inyectadas por Entorno/Configurador. | Cadenas literales (ej. `api_key = "sk-..."`) en repositorios o módulos Core. | P0 (Fallo de seguridad y migración imposible). | Uso de `os.getenv` + Keyring OS + Configuración Pydantic base. |
| ANT-004 | Confianza Implícita en LLM (Blind Trust) | El output crudo de un LLM se pasa directamente a una función `eval()`, `exec()`, o base de datos sin validación tipológica. | Ausencia de parser (e.g. `json.loads`) y validación (Pydantic) post-generación. | Crítico (Prompt Injection y destrucción de DB). | Validadores estrictos y CORTEX-TAINT obligatorio. |
| ANT-005 | Diseño de Interfaz Gorda (Fat Interface) | Una sola clase o módulo gestionando persistencia, lógica de red, embeddings y UI simultáneamente. | Archivos de código gigantes (>1000 líneas) con docenas de métodos no relacionados. | Alto (Inmantenibilidad y testeo imposible). | Segregar responsabilidades (SOLID), creación de `engine/`, `api/`, `storage/`. |
| ANT-006 | Acoplamiento a Terceros sin ACL | Dependencia directa y no abstraída de una SDK de proveedor en múltiples puntos de la lógica de negocio. | `import openai` esparcido en docenas de archivos. | Crítico (Vendor Lock-in, rotura por actualización). | Crear "Anti-Corruption Layer" (Wrappers o Facades unificados). |
| ANT-007 | Excepciones como Control de Flujo | Uso sistemático de bloques `try-except` para dirigir la lógica normal del programa en lugar de comprobaciones `if`. | `except KeyError: create_key()` usado cientos de veces por segundo. | Medio (Sobrecarga de VM Python, logs ruidosos). | Chequeos `if key in dict` o `dict.get()`. |
| ANT-008 | Espera Asíncrona Bloqueante (Sync-over-Async) | Llamar a una función de red o disco bloqueante (`requests.get`, `time.sleep`) dentro de una corrutina `async def`. | `time.sleep(1)` en un entorno FastAPI/asyncio. | Crítico (Congelamiento del Event Loop, latencia extrema). | Sustitución por `aiohttp` y `asyncio.sleep()`. |
| ANT-009 | Falta de Criterio de Finalización (Infinite Generation) | Agente LLM configurado en bucle sin condición rígida matemática para finalizar el ciclo de interacciones. | Loops iterativos que solo se detienen si el LLM dice "Stop". | Alto (Agotamiento de tokens y exergía térmica). | Imponer `Max_Iterations` y función de evaluación determinista de éxito. |
| ANT-010 | Delegación de Responsabilidad de Seguridad | Un módulo de aplicación confiando en que el front-end o un agente validará los parámetros de entrada. | APIs que asumen parámetros correctos sin validación Backend. | P0 (Bypass trivial de seguridad perimetral). | Re-validación estricta y "Zero Trust" en cada capa de interfaz. |
