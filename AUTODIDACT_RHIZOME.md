## 1. Isomorphic Extraction (Dejargonization)
*   **Rizoma (Rhizome):** A non-hierarchical, decentralized model of organization where any point can connect to any other point. -> *A non-directed, mesh-based graph of autonomous execution units and memory states with no single coordinating root node.*
*   **Mesetas (Plateaus):** Stable territories or groups of concepts within the rhizome. -> *Sub-graphs of highly-cohesive context, cached states, or specialized tool agents that maintain operational consistency without relying on a global schema.*
*   **Líneas de fuga / Desterritorialización (Lines of flight / Deterritorialization):** Points of rupture and escape that allow the system to mutate and form new connections. -> *Dynamic routing paths and run-time adaptation vectors that bypass static configuration to handle unforeseen context drift.*
*   **Calco vs. Mapa (Tracing vs. Mapping):** Tracing is a pre-determined reproduction of a structure; mapping is an open, interactive construction. -> *Static, rule-based workflow pipelines (tracing) vs. dynamic, self-organizing agent networks that construct their execution path at run-time (mapping).*

## 3. Structural Hole Detection
*   **Current Constraint (Arborescent Coordination):** The agent infrastructure still relies heavily on hierarchical delegation (e.g., Parent Agent -> Subagent -> Tool execution) where the Parent Agent acts as a single-point-of-failure coordinator and context bottleneck.
*   **Rhizomatic Solution (Mesh Routing & Event-Driven Mitosis):** If agents communicate via an asynchronous publish-subscribe message bus (where any agent can trigger, adapt, or listen to any message without a central manager), we eliminate the parent-child constraint. This transforms the swarm into a self-assembling mesh where the closest matching specialist claims a task.
