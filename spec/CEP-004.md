# CEP-004: Conformance Test Suite

**Status:** Frozen Normative Draft
**Layer:** Testing / Interoperability (C5-REAL)
**Authors:** BABYLON-60 Autonomous Design Swarm

---

## 1. Abstract

To guarantee that BABYLON-60 is truly a cross-platform protocol and not just a single reference implementation, absolute deterministic behavior must be verified. **CEP-004** defines the official Conformance Test Suite. Any implementation claiming to be a BABYLON-60 Microkernel **MUST** pass 100% of these test vectors.

---

## 2. The Conformance Philosophy

The microkernel is fundamentally a deterministic function `f(State_N, Proposal) -> State_N+1 | Panic`.

Therefore, conformance is tested not by asserting the language of implementation (Python, Rust, C++), but by executing standardized test vectors (JSON fixtures) and comparing the resulting cryptographic hashes and State IDs.

---

## 3. Test Vector Anatomy

A standard test vector is a self-contained JSON document containing:

1.  **`vector_id`**: Unique identifier for the test (e.g., `TV-001-CANONICAL-SORT`).
2.  **`description`**: The normative rule being tested.
3.  **`initial_state`**: The base DAG topology (can be empty).
4.  **`input_proposal`**: The `ProposedState` or raw objects to be committed.
5.  **`expected_behavior`**: `SUCCESS` or a specific Error Code (e.g., `ERR_CYCLE_DETECTED`).
6.  **`expected_hash`**: If `SUCCESS`, the exact `SHA3-256` hash or Merkle Root that the kernel MUST produce.

---

## 4. Test Categories

The suite is divided into the following categories to isolate failure domains:

### 4.1 Topology Vectors (TV-TOP)
Tests the kernel's ability to reject structurally invalid graphs.
*   **TV-TOP-010**: Reject Proposal with Dangling `evidence_id`. (`ERR_DANGLING_REFERENCE`)
*   **TV-TOP-020**: Reject Proposal forming a `DERIVED_FROM` causal loop. (`ERR_CYCLE_DETECTED`)

### 4.2 Serialization Vectors (TV-SER)
Tests the implementation of CEP-002 (Canonical Serialization).
*   **TV-SER-010**: Assert identical hash for Maps with identical content but varying input key order (tests Canonical Byte Ordering).
*   **TV-SER-020**: Assert CBOR integer compaction rules are applied before hashing.

### 4.3 State Machine Vectors (TV-STM)
Tests adherence to the `Proposed -> Validated -> Committed -> Archived` lifecycle.
*   **TV-STM-010**: Reject mutation of a `Committed` state.
*   **TV-STM-020**: Reject transition from `Committed` directly back to `Proposed`.

### 4.4 Cryptographic Vectors (TV-CRY)
Tests signature verification and capability evaluation.
*   **TV-CRY-010**: Reject valid topology if `AnnotationSet` signature is invalid. (`ERR_INVALID_SIGNATURE`)
*   **TV-CRY-020**: Reject write to namespace without capability. (`ERR_CAPABILITY_DENIED`)
