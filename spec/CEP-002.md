# CEP-002: Canonical Serialization & Hashing

**Status:** Frozen Normative Draft
**Layer:** Core Kernel / Data Representation (C5-REAL)
**Authors:** BABYLON-60 Autonomous Design Swarm

---

## 1. Abstract

For a multi-agent epistemic network to achieve cryptographic consensus, it is imperative that equivalent abstract data structures map to the exact same byte sequence across all implementations. CEP-002 defines the canonical serialization rules and hashing algorithms that the BABYLON-60 microkernel uses to compute identifiers, compute Merkle roots, and evaluate structural equality.

---

## 2. Canonical Serialization Formats

The kernel natively supports two canonical serialization formats, depending on the performance and language environment of the implementation. However, the primary byte-representation format used to generate the canonical ID is **Deterministic CBOR**.

1.  **Deterministic CBOR (RFC 8949):** The primary encoding. Preferred for Rust and C5-REAL core implementations due to density and speed.
2.  **Canonical JSON (RFC 8785):** A secondary text-based encoding. Used for debugging, web clients, and tooling interoperability.

*Regardless of the transport layer, an EpistemicObject's `ContentHash` MUST be computed from its Deterministic CBOR representation.*

---

## 3. Canonical Byte Ordering

Ambiguity in the order of elements destroys cryptographic integrity. All implementations MUST adhere to the following rules prior to serialization:

### 3.1 Unordered Collections
*   Any collection defined as a `Set` or `UnorderedList` in the abstract model MUST be sorted before it is serialized.
*   The sorting MUST be lexicographical based on the canonical byte sequence of the elements (byte-by-byte comparison).

### 3.2 Maps & Dictionaries
*   All keys in a Map MUST be of string or byte type.
*   Maps MUST be sorted by their keys lexicographically by bytes.

### 3.3 Timestamps & Numeric Types
*   Timestamps MUST be serialized as integer milliseconds since the Unix Epoch (UTC). Floating point time is strictly forbidden.
*   Integers MUST use the most compact representation permitted by Deterministic CBOR.

---

## 4. Hashing Protocol

The kernel relies on cryptographic hashes to generate Identifiers and Merkle DAG roots.

*   **Primary Hashing Algorithm:** `SHA3-256`.
*   **Derivation:** `ID = SHA3-256(canonical_serialize(Object))`
*   **Type Prefixing:** To prevent cross-type collisions, the byte sequence sent to the hashing algorithm MUST be prefixed with the object's type tag as a UTF-8 string, followed by a null byte `\0`.
    *   Example: `SHA3-256(b"Assertion\0" + CBOR_Bytes)`
