# Sovereign Worker MVP — Mac Maestro ↔ CORTEX
> **Objetivo:** Definir unidades mínimas de trabajo autónomo con ROI demostrable (C5-REAL), antes de auditar o expandir la integración arquitectónica.

---

## Workflow 1: Extracción de Leads de Alto Valor (B2B SOTA)

```yaml
Input:
  - Lista cruda de 50 dominios (ej. YC startups 2026).
  - Criterio de cualificación (ej. "Tienen pricing visible y usan React").

Action (Mac Maestro + CORTEX):
  1. Mac Maestro abre el navegador y visita cada URL.
  2. Extrae el DOM / visión de la web y busca señales (precios, stack, equipo).
  3. Swarm evalúa la cualificación en paralelo.
  4. CORTEX almacena un hecho estructurado para los leads cualificados.

Output:
  - Dataset consolidado de leads enriquecidos en `cortex.db` con root causal.
  - Exportación a CSV/Markdown.

Time:
  - < 15 minutos de forma autónoma.

Value:
  - Reemplaza 4h de prospección SDR manual.
  - Genera un pipeline de ventas validado empíricamente.
```

## Workflow 2: Auditoría Visual de UI (Regression Testing)

```yaml
Input:
  - URL del entorno de staging (ej. `cortexpersist-com.vercel.app`).
  - Base de conocimiento CORTEX con los "Aesthetic Principles" (Industrial Noir 2026).

Action (Mac Maestro + CORTEX):
  1. Mac Maestro navega por las 5 rutas críticas del producto.
  2. Captura screenshots completos y los envía al Swarm.
  3. El Swarm audita alineación cromática, contraste, responsive y LCP.
  4. CORTEX registra las desviaciones como "issues" y alerta si hay roturas de diseño (L-EPI Guard).

Output:
  - Reporte de auditoría visual con coordenadas precisas del fallo en el DOM.

Time:
  - < 3 minutos pre-deploy.

Value:
  - Aniquila el QA manual visual.
  - Asegura que ninguna actualización rompa la estética (Zero-Drift).
```

## Workflow 3: Síntesis del Estado del Arte (SOTA Radar)

```yaml
Input:
  - Palabra clave o paper (ej. "Takens Delay Embedding for LLMs").

Action (Mac Maestro + CORTEX):
  1. Mac Maestro abre ArXiv, GitHub y Twitter/X.
  2. Ejecuta búsquedas, descarga PDFs y lee repositorios en tendencia.
  3. Swarm sintetiza los findings y falsifica claims inútiles.
  4. CORTEX cristaliza el SOTA en su memoria a largo plazo (Hologram Memory).

Output:
  - Documento SOTA destilado con referencias exactas, listo para implementar.

Time:
  - < 20 minutos (autónomo en background).

Value:
  - Reemplaza 1-2 días de investigación humana (Lectura rápida, filtrado de ruido).
  - Mantiene a la organización siempre en la frontera (SOTA).
```

## Workflow 4: Triaje Autónomo de GitHub Issues

```yaml
Input:
  - URL del repositorio y listado de Issues abiertos.

Action (Mac Maestro + CORTEX):
  1. Mac Maestro navega por los issues.
  2. Recupera la base de código desde CORTEX Memory para entender contexto.
  3. Swarm evalúa si es un bug reproducible, pide info extra si falta, o redacta PR propuesta.
  4. Mac Maestro escribe la respuesta en el navegador y le da a "Comment".

Output:
  - Issues triados, etiquetados, y contestados con precisión técnica.

Time:
  - ~1 minuto por issue.

Value:
  - Reemplaza horas de contexto mental para el mantenedor.
  - Reduce el "Time-to-Response" a cero.
```
