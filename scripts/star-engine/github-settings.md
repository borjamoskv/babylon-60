# 🔧 GitHub Settings — Manual Configuration Checklist

These settings must be configured manually in GitHub's web UI.

## 1. Social Preview Image

1. Go to https://github.com/borjamoskv/Cortex-Persist/settings
2. Scroll to "Social Preview"
3. Upload `assets/marketing/social-preview.png`
4. Save

**Verify:** Paste your repo URL in https://opengraph.dev and confirm the preview shows

## 2. Repository Topics (max 20)

Add these topics in the repo settings (About section → gear icon):

```
ai-agents
tamper-evident
memory
audit-trail
cryptography
sha256
merkle-tree
llm
ai-safety
compliance
python
trust
observability
multi-agent
append-only-log
decision-lineage
forensics
autonomous-agents
ai-infrastructure
developer-tools
```

## 3. Repository Description

Set the description to:
```
Tamper-evident memory & decision lineage for AI agents. Cryptographic proof of what your agent knew.
```

## 4. Website URL

Set to: `https://cortexpersist.com` (or your docs URL)

## 5. Enable GitHub Discussions

1. Go to Settings → Features
2. Enable "Discussions"
3. Create these categories:
   - 📢 **Announcements** (only maintainers post)
   - 💡 **Ideas** (feature requests from community)
   - 🙏 **Q&A** (support questions)
   - 🙌 **Show and Tell** (community projects using CORTEX)
   - 💬 **General** (open discussion)

## 6. Good First Issues

Label at least 3-5 issues with `good first issue` to attract contributors.
Ideal candidates:
- Documentation improvements
- Adding more examples
- Writing additional tests
- Improving error messages
- CLI UX enhancements
