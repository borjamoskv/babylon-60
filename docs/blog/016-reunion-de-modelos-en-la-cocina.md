# Reunión de Modelos en la Cocina

Imagina una cocina a las ocho de la mañana. Entra el modelo rápido, deja un vaso
en cualquier sitio y dice: "yo te arreglo esto en diez segundos". Entra el modelo
grande, mira la encimera, pide silencio y empieza a diseñar una teoría general
del desayuno. Entra el modelo local, abre la nevera y pregunta por qué vamos a
pedir comida fuera si hay ingredientes suficientes.

Ninguno está equivocado. El problema es pedirles la misma cosa.

## El rápido

El modelo rápido es perfecto para tareas pequeñas:

- renombrar variables;
- clasificar tickets;
- resumir logs no sensibles;
- detectar formato;
- proponer un primer borrador.

No le pidas filosofía de migraciones. Te dará una respuesta con mucha confianza
y poca memoria del incendio.

## El grande

El modelo grande entra cuando hay ambigüedad real:

- arquitectura;
- seguridad;
- decisiones irreversibles;
- debugging largo;
- estrategia de producto;
- revisión de tradeoffs.

No hace falta despertarlo para cambiar una coma. Es como usar una sala de juntas
para decidir dónde poner una taza.

## El local

El modelo local tiene una virtud menos brillante y más importante: está ahí.

No se queja de cuota. No pide mandar contexto privado fuera. No desaparece si la
red está rara. Puede no ser el más listo de la sala, pero para triage, borradores,
operación rutinaria y privacidad, su constancia vale mucho.

## El router como jefe de cocina

El router decide quién toca cada plato.

```text
tarea simple -> rapido
datos sensibles -> local
decision critica -> grande
revision final -> agente auditor
```

El caos empieza cuando todo va al mismo modelo por costumbre. O cuando se elige
modelo por prestigio, no por ajuste.

## La memoria es la receta

Sin memoria compartida, cada modelo entra en la cocina y pregunta dónde están los
cuchillos. Con memoria persistente, todos leen la receta, ven qué se quemó ayer y
qué ingrediente no se debe usar.

El modelo cambia. La receta permanece.

## La tesis

No necesitas un modelo perfecto. Necesitas una brigada bien asignada.

El rápido corta verduras. El grande decide el menú. El local cuida la despensa.
El router evita que todos intenten freír lo mismo.

Así empieza a parecer una cocina.
