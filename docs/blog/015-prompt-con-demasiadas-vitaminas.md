# Tu Prompt Tiene Demasiadas Vitaminas

Hay prompts que no nacen. Se amontonan.

Empiezan con una instrucción razonable: "actúa como ingeniero senior". Luego
alguien añade "sé creativo". Después "sé conciso". Más tarde "explica todo con
detalle". Al final el agente recibe una tortilla de mandatos donde debe ser breve,
exhaustivo, agresivo, prudente, poético, exacto, veloz y humilde, todo en la
misma frase.

El resultado no es inteligencia. Es gimnasia semántica.

## El prompt multivitamínico

El prompt inflado tiene síntomas claros:

- usa diez adjetivos donde bastaba un verbo;
- pide autonomía y permiso para cada paso;
- exige creatividad pero penaliza el riesgo;
- solicita máxima profundidad en tres líneas;
- mezcla tono, arquitectura, calendario y terapia;
- contiene reglas que se contradicen sin admitirlo.

El agente no falla porque sea torpe. Falla porque le has pedido que frene,
acelere y aparque al mismo tiempo.

## Menos perfume, más contrato

Un buen prompt no impresiona. Orienta.

Debe decir:

1. qué objetivo importa;
2. qué restricciones no se pueden cruzar;
3. qué evidencia cuenta como terminado;
4. qué hacer si falta información;
5. qué estilo de respuesta prefiere el usuario.

Todo lo demás debería ganarse su sitio.

La frase "sé excelente" no añade nada. La frase "antes de tocar ledger, ejecuta
la verificación de continuidad" sí.

## El prompt no es memoria

Otro error clásico es usar el prompt como trastero. Se meten preferencias, reglas,
historial, bugs, comandos, enlaces, advertencias y alguna promesa hecha en marzo.

Eso no es memoria. Es una mochila sin cremalleras.

La memoria persistente existe para sacar peso del prompt. El prompt debería
orientar la sesión actual. La memoria debería conservar lo que debe sobrevivir a
la sesión.

Si todo vive en el prompt, el agente no recuerda. Carga.

## La dieta mínima

Antes de añadir otra línea, pregunta:

> ¿Esta instrucción cambia una decisión real del agente?

Si no, fuera.

Un prompt sano cabe en una estructura simple:

```text
Rol
Objetivo
Restricciones
Criterio de exito
Estilo
```

Después, que el sistema haga su trabajo: herramientas, memoria, tests, permisos,
logs y revisión.

## La tesis

El prompt perfecto no es el más largo. Es el que deja al agente sin excusas y sin
ruido.

Un prompt vitaminado promete poder.

Un prompt bien diseñado produce trabajo.
